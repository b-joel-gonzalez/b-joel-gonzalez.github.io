Introduction to Computer Systems

CIS240/CIT593

An introductory course on computer systems. Students learn the bottom-up fundamentals of modern computers, from CMOS transistor circuits to digital logic basics. Once they understand how datapaths and ISAs work, they then switch to a top-down view and work in assembly and C, building their understanding of an operating system.

Course projects include writing a basic user/OS program in LC-4 assembly (an educational programming language similar to LC-3) and writing a disassembler and a compiler in C.

CIS240 (Fall/Spring) is the undergraduate version while CIT593 (Summer) is the graduate version, but they shared the same course material.

Operating Systems

CIS380/CIS548/CIT595

A project course on operating systems. This course builds upon the latter concepts introduced in CIS240/CIT593. Students learn about the basics of an operating system, including process management, system calls, memory management, scheduling algorithms, file systems, and more.

Students mainly work with Unix-like systems and program in C. Course projects include writing a user shell and building "PennOS", a Unix-like operating system with a priority scheduler and flat filesystem.

CIS380 (Fall) is the undergraduate version while CIS548 (Spring) and CIT595 (Summer) are the graduate versions, the latter belonging to the MCIT program. CIS548 had an additional project on memory management while CIT595 cut out the PennOS project and instead had a project on networked systems.

Computer Architecture

CIS371/CIS501

A project course on computer architecture. This course builds upon the earlier concepts introduced in CIS240/CIT593. Students learn more about how modern processors are optimized for speed and efficiency. Methods discussed include pipelining, superscalar design, branch prediction, out-of-order execution, caching, multicore design, and more.

Throughout the course, students use Verilog to program digital logic building blocks such as a ripple-carry adder, a carrylook-ahead adder, a divider, and an ALU. They then take these building blocks to build up an LC-4 CPU, enhanced with pipelining and superscalar designs. Students simulate their designs using Vivado and upload their code to a ZedBoard FPGA.

CIS371 (Spring) is the undergraduate version while CIS501 (Fall) is the graduate version, but they shared the same course material.

Silicon Garage

ESE190

An introduction to hardware/software lab work using the Arduino platform - the course is designed for non-engineers who don't have much programming or circuit experience. Students learned the basics of programming and physics to build their own circuit projects using Arduino.

Students used the standard Arduino Uno model for the labs. Labs included turning on LEDs with pushbuttons, building a mini-thermostat, and using WiFi for IoT communication. They wrote their code in C and were encouraged to use external libraries as needed for their final project.

Here is the ESE190 website.

Embedded Systems

ESE350/519

A project lab course exploring embedded systems designs, implementations, and applications. The course starts from the hardware-level and covers power management and circuitry for an embedded system. It then moves to low-level programming, understanding how to write C to control the various pins and registers on an embedded platform. It then finishes with a conceptual, software-level tour of real-time operating systems before students build a final project of their choosing.

Students wrote their code in C on the Arduino Uno, using the standard C + AVR + FreeRTOS libraries for the labs and final project. For the final project, they were allowed to use other microcontrollers if preferred.

ESE350 (Spring) is the undergraduate version while ESE519 (Fall) is the graduate version, but they largely shared the same material.

Senior Design Capstone

ESE450/451

A senior project capstone course for the Department of Electrical and Systems Engineering. Students in electrical engineering, systems engineering, computer engineering, network engineering, and more take this two-part course to build a project of their own choosing. Students frequently give updates on their progress through staff meetings, written reports, and class presentations, culminating in a demo day finale + final report/video at the end of both semesters.

There is always a wide variety in the projects that we see for ESE Senior Design, ranging from pure software to heavy hardware to a mix of both. I've had the joy of mentoring both the Class of 2020 + Class of 2021 in ESE Senior Design and I'm consistently impressed by the effort and creativity of my students.

TeaBot

ESE450/451: Senior Design Capstone

I worked in a team of four to develop a robot that could harvest tea leaves. This project was a combination of computer vision and mechatronics. Our robot used an Intel RealSense depth camera to identify a leaf using image processing and then direct a custom-manufactured arm to move to extract the leaf.

We were able to get a basic proof-of-concept model working by the end of the year, which could hopefully be built upon in the future. Here is my team's final report along with our Devpost submission which has a video summary.

BattleBot

MEAM510: Mechatronic Systems

I worked in a team of four to develop a robot that could participate in a team tower defense game along with several other robots. Our team built the robot from scratch, laser-cutting the pieces from CAD models to assemble, and was fitted with a swinging arm to attack other robots. The robot navigated using 4-wheel differential drive and was powered by AA/9V/LiPo batteries. We wrote C++ code on ESP32 and ATmega32 boards, directing the robot using a custom remote controller.

This was a very challenging but incredibly fun project that took over a month of planning and effort! Here is my team's final report along with a video demo of our robot on the field.

Digital Audio Filter

PHYS364: Laboratory Electronics

I worked in a team of two to develop a system to filter and separate out treble and bass from an audio signal, using the filtered outputs to light up respective LED strips. We implemented the high-pass/low-pass filtering in hardware, fed the filtered inputs into an Arduino to detect whether there was treble or bass, and then used the Arduino to drive the appropriate BJT to light up the LEDs.

This was a project that we did for extra credit on top of the regular labs at the end of the course, and was a creative finish to a wonderful course. Here is my team's final report.

SRAM Design

ESE370: Digital Circuits

I worked in a team of two to develop an SRAM array at the MOSFET-level. The SRAM array was able to hold 16 4-bit data elements and worked as a FIFO queue. The FIFO queue could take in an enqueue or dequeue signal which would either insert a new 4-bit data element (if there was space) into the array or read out the next 4-bit data element in the array (if it was present) respectively. We had to design the SRAM system in two parts: the actual array with 16x4 SRAM cells (6 MOSFETs each), and the control logic that took in a clock signal, enqueue signal, and dequeue signal and directed the system as needed.

A very challenging but engaging project in digital design, built using Electric and tested using Ngspice. Unfortunately, I'm unable to publically share my report for this project in the interest of future students taking the course. But if you're curious, here is the course's lab manual.

Ripple-Carry Adder

ESE370: Digital Circuits

I developed an 8-bit ripple-carry adder (RCA) at the MOSFET-level. I built the RCA from the bottom-up, cascading several full-adders together to form an 8-bit adder. The goal of the project was to optimize for delay - we were allowed to toggle several parameters in order to achieve this metric. This included gate structure, gate type, MOSFET sizes, logic discipline, and supply voltage.

Using Electric and Ngspice, this project tied together various concepts about optimization in digital design that we discussed in the course. Unfortunately, I'm unable to publically share my report for this project in the interest of future students taking the course. But if you're curious, here is the course's lab manual.

Fall 2019

Pipelined/Superscalar CPU

CIS501: Computer Architecture

I worked in a team of two to develop a 5-stage pipeline + 2-way superscalar processor using Verilog. We started from the very basic building blocks, writing a half-adder, then a full-adder, then a ripple-carry adder. We then improved upon this design to build a carry-lookahead adder, which we integrated into an Arithmetic Logic Unit (alongside a custom divider). Once we had the ALU coded, we moved onto the entire LC-4 datapath, which we enhanced with pipelining and superscalar design.

It was very satisfying to build up the entire datapath from basic logic gates, and a valuable experience to work with Verilog and FPGAs. Unfortunately, I'm unable to publically share my code for this project in the interest of future students taking the course. But if you're curious, here is the course's repository, where you can see the skeleton code for all the labs.
